Spring IOC

  1. 什么是IOC？

这个大家已经非常熟悉了，控制反转。就是将我们的Bean交给Spring容器来控制。

  1. 如何使用IOC？

    • 自动注入
    // 实体类
@Data
public class Student {
	private int id;
	private String name;
}
    // Repository
@Repository
@Mapper
public class StudentMapper extends BaseMapper<Student>{
	public User findById(int id);
}
    // Service
@Service
public class StudentService {

  // 自动注入
	@Autowired
	StudentMapper stuMapper;
	
	public User findById(int id) {
		return stuMapper.findById(id);
	}
}
    // 主类
@SpringBootApplication
// 扫描注入的包
@MapperScan("com.auto.dao.*")
public class SpringBootMain {

      public static void main(String[] args) {  
       SpringApplication.run(App.class, args);  
      }  
}
    • 手动set方式注入
    // 实体类
@Data
public class Student {
	private int id;
	private String name;
}
    // Repository
@Repository
@Mapper
public class StudentMapper extends BaseMapper<Student>{
	public User findById(int id);
}
    // Service
@Service
public class StudentService {

	StudentMapper stuMapper;
	
	public void setStuMapper(StudentMapper stuMapper) {
		this.stuMapper = stuMapper;
	}
	public User findById(int id) {
		return stuMapper.findById(id);
	}
}
    // config类
@Configuration
public class config {

  // 手动注入Bean
	@Bean
	public StudentMapper mapper() {
		return new StudentMapper();
	}
	
	@Bean
	public StudentService stuService() {
		StudentService s = new StudentService();
		StudentMapper sM = new StudentMapper();
		s.setStuMapper(sM);
	}
}

    以上代码为两种方式的注入方式。

    Spring AOP

    1. aop是什么？

    通俗易懂的说就是面向切面，那问题又来了，什么是面向切面？

    如下图所示，将程序从中间切开后，植入一些自己的逻辑，就是面向切面。

    1. 面向切面的维度有哪些？

      • 切点（pointcut）

        切点就是告知Spring框架，我们要切入方法，但是Spring框架不知道我们要切入哪一个方法，这里就引入了一个 切入点（execution） 的名词。

        • 切入点（execution）

          切入点就是让Spring框架知道我们要从哪个方法切入。我们知道切入方法了，但是我想在切入方法前、后或者前后都执行一些其他验证或方法，那就要引入一个 增强（advice） 概念。

          • 增强（advice）
            • 前增强
            • 后增强
            • 方法执行成功后增强
            • 环绕增强
            • 后置异常增强
- 切面（aspect）

  切面就是告知Spring框架，我们要切开了，这里就是一个面。

  *<u>一个切面可以有多个切点，一个切点上面有多个切入点。</u>*
